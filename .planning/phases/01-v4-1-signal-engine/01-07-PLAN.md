  <action>
  Implement backend/interfaces/rest/routers/signals.py following the existing router pattern (see backtests.py). APIRouter(prefix="/api/v1/signals", tags=["signals"]). Three endpoints:

  GET "/" → response_model=list[SignalVector]: query crypto_universe for active coins, call signal_service.evaluate() for each coin using latest available data from DB, return list. Use async DB session via Depends(get_async_session). Cache result for 1 hour (use simple in-memory dict with timestamp — no Redis needed).

  GET "/{coin}" → response_model=SignalVector: coin is symbol like "BTC-USD". 404 if not in crypto_universe. Calls signal_service.evaluate() for that coin. Returns SignalVector.

  GET "/backtest/{coin}" → response_model=BacktestReport (note: prefix is /api/v1/signals, so this resolves to /api/v1/signals/backtest/{coin} — alternatively mount on /api/v1 and use separate prefix /backtest). Use path GET /api/v1/backtest/{coin} with a separate APIRouter(prefix="/api/v1/backtest", tags=["backtest"]). Calls walkforward.run_walkforward() on stored price data and latest signals. Returns BacktestReport.

  Register both routers in backend/interfaces/rest/app.py via app.include_router(signals_router) and app.include_router(backtest_router). Do NOT modify existing router registrations.
  Commit: feat(api): read-only signals + backtest endpoints (A7.9)
  </action>
  <verify>curl http://localhost:8000/api/v1/signals (dev server running) returns JSON list; /api/v1/signals/BTC-USD returns single SignalVector; /api/v1/backtest/BTC-USD returns BacktestReport</verify>
