    TestNoLookAhead (D-09):
    - build_signals(close, ma_window=50) on a strictly monotone increasing series
    - signals.shift(-1) should correlate >= 0.999 with close (signals ARE look-ahead if not shifted)
    - signals itself must NOT correlate >= 0.999 with close (shift was applied correctly)
    - Alternatively: for day i, signal[i] must depend on close[i-1], not close[i]; verify by checking that first non-NaN signal comes from day >=1 and that close[0] > sma[0] does NOT necessarily produce signal[0]=1

    TestCostSensitivity (D-03):
    - Create synthetic prices+signals; call run_cost_sensitivity() with monkeypatched _download returning synthetic data
    - Assert results list has len == 2*3 == 6 (2 coins × 3 costs)
    - Assert cost=0.005 produces lower strategy_sharpe than cost=0.001 for same coin (higher costs = lower net return)
    - Assert each result.cost_level in [0.001, 0.002, 0.005]

    TestRegimeSplit (D-04):
    - Directly call the internal _run_regime logic (or expose a testable helper from the script)
    - Using synthetic prices from 2015-01-01 to 2018-12-31 (>315 rows), run with regime_start="2018-01-01", regime_end="2018-12-31"
    - Assert regime_net index min >= "2018-01-01" and max <= "2018-12-31"
    - Assert oos_rows > 0 (OOS points exist within regime window because full history was used as training)

    TestUniversumInsufficient (D-05):
    - Create a mock where _download returns None for "AVAX-USD" (simulate no data)
    - Call run_universe() with monkeypatched downloads
    - Assert "AVAX-USD" result is a dict with "status" == "insufficient" (or "download_failed")
    - Coins with real data return UniverseResult (not dict)

    TestParameterStability (D-06):
    - Use synthetic close (trending 800-row series, seed=42)
    - run_parameter_stability() with monkeypatched _download
    - Assert results cover all 5 windows [50, 75, 100, 150, 200]
    - Assert exactly one result has is_default==True (window==100)
    - Assert results for window=50 and window=200 differ in strategy_sharpe (different MA windows produce different results)

    TestBuyAndHold (D-07):
    - Call _bah_metrics(close) with synthetic 600-row trending series
    - Assert returned tuple is (float, float) — (sharpe, calmar)
    - Assert sharpe > 0 for trending series (B&H on uptrend should be positive)
    - Verify avg_exposure ≈ 1.0: call run_walkforward_with_details(prices_df, pd.Series(1.0, index=close.index), costs=0.0001) directly and assert details["avg_exposure"] > 0.99
