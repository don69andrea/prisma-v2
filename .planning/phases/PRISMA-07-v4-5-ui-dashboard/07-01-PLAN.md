# Plan 07-01: Backend API Endpoints for V4-5 Dashboard

## Objective

Add three read-only or append-only REST endpoints required by the V4-5 UI:

1. `GET /api/v1/agent-audit/{coin}` — returns the full `agent_run` JSONB from `agent_audit_trail` for a coin (most recent entry), typed as `AgentAuditResponse`
2. `GET /api/v1/ohlcv/{coin}?days=120` — returns OHLCV data for a coin via `CryptoPriceAdapter`, for candlestick + indicator charts
3. `POST /api/v1/signals/{coin}/confirm` — persists HITL decision (proceed/abort) to `hitl_confirmations` table (append-only, no auto-trading)

## Context

- `agent_audit_trail` table: has `id`, `coin`, `asof`, `agent_run` (JSONB), `created_at`. Append-only repository exists at `backend/infrastructure/persistence/repositories/agent_audit_trail_repository.py`.
- `AgentAuditTrailRepository` has only `insert()`. Need to add a `find_latest_by_coin()` method.
- `CryptoPriceAdapter.fetch_ohlcv()` at `backend/infrastructure/adapters/crypto_price_adapter.py` — fetches via yfinance. Use `asyncio.to_thread()`.
- No new table for OHLCV — data comes from yfinance directly.
- HITL confirm requires a new table `hitl_confirmations` (migration 0043).
- Latest migration: `0042_widen_news_source_column.py`.
- Iron rules (AGENTS.md): no `any` types, Pydantic-validated, no auto-trading logic.

### Task 4: AgentAuditTrailRepository — add find_latest_by_coin()

In `backend/infrastructure/persistence/repositories/agent_audit_trail_repository.py`, add:
```python
async def find_latest_by_coin(self, coin: str) -> AgentAuditTrailORM | None:
    """Returns the most recent audit trail entry for a coin (by created_at DESC)."""
    result = await self._session.execute(
        sa.select(AgentAuditTrailORM)
        .where(AgentAuditTrailORM.coin == coin.upper())
        .order_by(AgentAuditTrailORM.created_at.desc())
        .limit(1)
    )
    return result.scalar_one_or_none()
```

### Task 5: REST router — 3 endpoints

**Endpoint 1:** `GET /api/v1/crypto/{coin}/agent-audit`
```python
@router.get("/{coin}/agent-audit", response_model=AgentAuditResponse)
async def get_agent_audit(coin: str, audit_repo=Depends(get_audit_trail_repo)):
    row = await audit_repo.find_latest_by_coin(coin.upper())
    if row is None:
        raise HTTPException(404, detail=f"No audit trail found for {coin}")
    # Parse agent_run dict into AgentRunDetail (use model_validate)
    agent_run = AgentRunDetail.model_validate(row.agent_run)
    return AgentAuditResponse(
        audit_trail_id=row.id, coin=row.coin, asof=row.asof,
        agent_run=agent_run, created_at=row.created_at
    )
```

**Endpoint 2:** `GET /api/v1/crypto/{coin}/ohlcv?days=120`
```python
@router.get("/{coin}/ohlcv", response_model=OHLCVResponse)
async def get_ohlcv(coin: str, days: int = Query(default=120, ge=7, le=365)):
    symbol = f"{coin.upper()}-USD"
    start = (date.today() - timedelta(days=days)).isoformat()
    adapter = CryptoPriceAdapter()
    df = await adapter.fetch_ohlcv(symbol=symbol, start=start)
    bars = [
        OHLCVBar(date=row.date, open=row.open, high=row.high,
                 low=row.low, close=row.close, volume=row.volume)
        for _, row in df.iterrows()
    ]
    return OHLCVResponse(coin=coin.upper(), symbol=symbol, bars=bars)
```
Validate `coin` against `_CRYPTO_UNIVERSE` allowlist. Return 404 if not in universe.
Use `asyncio.to_thread` via `CryptoPriceAdapter.fetch_ohlcv()` which already does this.

**Endpoint 3:** `POST /api/v1/crypto/{coin}/confirm`
```python
@router.post("/{coin}/confirm", response_model=HitlConfirmResponse, status_code=201)
async def confirm_hitl(coin: str, body: HitlConfirmRequest, repo=Depends(get_hitl_repo)):
    # Read-only UI: this only LOGS the decision, never triggers trades
    new_id = await repo.insert(
        audit_trail_id=body.audit_trail_id,
        coin=coin.upper(),
        decision=body.decision,
    )
    return HitlConfirmResponse(
        id=new_id, audit_trail_id=body.audit_trail_id,
        coin=coin.upper(), decision=body.decision,
        decided_at=datetime.now(tz=UTC),
    )
```
